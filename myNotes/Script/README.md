# 📥 sync-from-teacher.sh

Скрипт `sync-from-teacher.sh` нужен для синхронизации файлов из репозитория преподавателя в ваш репозиторий.

Он автоматически:

- 🔗 подключает репозиторий преподавателя как дополнительный `remote`
- 📡 загружает изменения из ветки преподавателя
- 📂 копирует файлы в ваш репозиторий
- ♻️ заменяет совпадающие файлы на актуальные версии
- 🛟 создает резервную ветку перед началом работы
- ✅ делает `commit`
- 🚀 отправляет изменения в ваш GitHub-репозиторий

---

## 📌 Для чего нужен этот скрипт

Этот скрипт полезен, если преподаватель обновляет проект в своем репозитории, а вам нужно быстро перенести эти изменения в свой репозиторий без ручного копирования файлов.

То есть вместо того, чтобы:

- самому скачивать архив
- искать, какие файлы изменились
- вручную заменять их
- бояться что-то потерять

можно просто запустить один скрипт.

---

## 🧭 Какие репозитории используются

### Репозиторий преподавателя
- ветка: `master`

### Ваш репозиторий
- ветка: `main`

Скрипт берет файлы из репозитория преподавателя и переносит их в ваш локальный репозиторий, после чего делает `push` в ваш `origin`.

---

## 📁 Где должен находиться скрипт

Скрипт должен лежать **внутри вашего локального репозитория**.

Пример структуры:

```bash
MfuaGit/
├── .git/
├── myNotes/
│   └── Script/
│       └── sync-from-teacher.sh
├── src/
├── README.md
└── ...

В вашем случае можно хранить его здесь:
myNotes/Script/sync-from-teacher.sh

Это удобно, потому что скрипт сам определяет корень репозитория по своему местоположению.
⚙️ Что делает скрипт по шагам
При запуске скрипт выполняет следующие действия:
1. Проверяет, что git установлен
Если git не найден, скрипт завершится с ошибкой.
2. Определяет корень репозитория
Скрипт ищет .git относительно папки, где лежит сам файл sync-from-teacher.sh.
3. Проверяет текущую ветку
Скрипт ожидает, что вы находитесь на ветке:
main

Если активна другая ветка, выполнение остановится.
4. Проверяет, что нет незакоммиченных изменений
Если в репозитории есть несохраненные изменения, скрипт не продолжит работу, чтобы случайно ничего не затереть.
5. Создает резервную ветку
Перед синхронизацией автоматически создается backup-ветка, например:
backup-before-teacher-sync-20260330-123456

Это страховка на случай, если вы захотите откатиться назад.
6. Подключает репозиторий преподавателя
Скрипт добавляет или обновляет remote с именем:
teacher

7. Загружает изменения из репозитория преподавателя
Используется ветка:
master

8. Копирует файлы преподавателя в ваш репозиторий
Если файлы уже существуют, они заменяются.
9. При необходимости удаляет лишние файлы
Только если вы явно запускаете скрипт с флагом:
--delete-missing

10. Делает commit и push
Если после синхронизации есть изменения, скрипт создает коммит и отправляет его в ваш репозиторий.
▶️ Как запустить
1. Сделать скрипт исполняемым
chmod +x myNotes/Script/sync-from-teacher.sh

2. Запустить обычную синхронизацию
./myNotes/Script/sync-from-teacher.sh

🧪 Режимы работы
1. Обычный режим
./myNotes/Script/sync-from-teacher.sh

Что делает:
копирует файлы из репозитория преподавателя
заменяет совпадающие файлы
не удаляет файлы, которых нет у преподавателя
Когда использовать:
Когда вы хотите просто обновить проект, но сохранить свои дополнительные файлы.
2. Режим с удалением лишних файлов
./myNotes/Script/sync-from-teacher.sh --delete-missing

Что делает:
копирует файлы из репозитория преподавателя
заменяет совпадающие файлы
удаляет tracked-файлы, которых нет в репозитории преподавателя
Важно:
Этот режим не удаляет untracked-файлы.
То есть удаляются только те файлы, которые уже отслеживаются git.
Когда использовать:
Когда нужно максимально приблизить ваш репозиторий к состоянию репозитория преподавателя.
❗ Важные условия перед запуском
Перед запуском убедитесь, что:
📍 вы находитесь на ветке main
💾 у вас нет незакоммиченных изменений
📂 скрипт лежит внутри вашего локального репозитория
🌐 у вас есть доступ к интернету для git fetch и git push
🛡️ Почему создается backup-ветка
Перед началом синхронизации скрипт создает резервную ветку.
Это сделано специально для безопасности.
Если после синхронизации результат вам не подойдет, можно вернуться к предыдущему состоянию.
Пример названия:
backup-before-teacher-sync-20260330-123456

Посмотреть список веток можно так:
git branch

🔄 Пример типичного использования
Ситуация:
Преподаватель обновил проект, и вам нужно перенести изменения в свой репозиторий.
Действия:
chmod +x myNotes/Script/sync-from-teacher.sh
./myNotes/Script/sync-from-teacher.sh

Результат:
файлы преподавателя будут скопированы в ваш проект
измененные файлы заменятся
будет создан commit
изменения отправятся в ваш GitHub
🧨 Что делать, если скрипт не запускается
Ошибка: git is not installed or not in PATH
Причина:
Git не установлен или недоступен из терминала.
Решение:
Проверьте:
git --version

Если команда не работает, установите Git.
Ошибка: This script must be placed somewhere inside your local git repository
Причина:
Скрипт лежит вне папки вашего репозитория.
Решение:
Переместите его внутрь проекта, например:
MfuaGit/myNotes/Script/sync-from-teacher.sh

Ошибка: You are on '...' Switch to 'main' first
Причина:
Вы находитесь не на ветке main.
Решение:
git checkout main

Ошибка: Working tree is not clean
Причина:
В репозитории есть незакоммиченные изменения.
Решение:
Сначала нужно:
либо закоммитить изменения
либо убрать их
либо сохранить через stash
Проверить состояние можно так:
git status

📚 Полезные команды
Проверить текущую ветку
git branch

Проверить состояние репозитория
git status

Посмотреть remote-репозитории
git remote -v

Посмотреть backup-ветки
git branch

✅ Итог
Скрипт sync-from-teacher.sh автоматизирует перенос файлов из репозитория преподавателя в ваш репозиторий и делает этот процесс:
быстрее ⚡
безопаснее 🛟
удобнее 👍
понятнее 📘
Он особенно полезен, когда нужно регулярно обновлять проект по версии преподавателя и не делать это вручную каждый раз.
👨‍💻 Примечание
Скрипт предназначен для учебной работы с двумя репозиториями:
репозиторий преподавателя — источник файлов
ваш репозиторий — место, куда изменения переносятся
По умолчанию скрипт работает осторожно и не удаляет лишние файлы, если вы сами не укажете флаг --delete-missing.
