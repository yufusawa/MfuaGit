# 📂 Скрипты для синхронизации репозиториев

Небольшой набор вспомогательных скриптов для синхронизации файлов из репозитория преподавателя в мой собственный репозиторий.

Эти скрипты решают одну простую задачу:

➡️ **копировать последние файлы преподавателя в мой репозиторий**  
➡️ **при необходимости удалять старые отслеживаемые файлы, которых больше нет в репозитории преподавателя**  
➡️ **безопасно коммитить и отправлять изменения**

---

## ✨ Что входит в набор

В этой папке находятся две версии одного и того же инструмента:

- `sync-from-teacher.sh` — для **Bash**
- `sync-from-teacher.ps1` — для **PowerShell**

Оба скрипта выполняют одну и ту же задачу.

---

## 🎯 Назначение

Скрипты синхронизируют файлы из Git-репозитория преподавателя:

- **Репозиторий преподавателя:** `https://gitflic.ru/project/rurewa/mfua.git`
- **Ветка преподавателя:** `master`

в мой собственный Git-репозиторий:

- **Мой репозиторий:** `https://github.com/yufusawa/MfuaGit.git`
- **Моя ветка:** `main`

---

## 💡 Зачем я это сделал

Я создал эти скрипты, чтобы не копировать файлы между репозиториями вручную и сделать обновление быстрее, безопаснее и удобнее для повторного использования.

---

## ⚙️ Основное поведение

По умолчанию скрипты:

✅ получают изменения из репозитория преподавателя  
✅ копируют и заменяют совпадающие файлы в моём репозитории  
✅ сохраняют дополнительные файлы, которые есть только в моём репозитории  
✅ создают резервную ветку перед внесением изменений  
✅ коммитят и отправляют результат в `origin/main`

При включении режима удаления скрипты также:

⚠️ удаляют **отслеживаемые** файлы из моего репозитория, которых больше нет в репозитории преподавателя

---

## 🛡️ Механизмы безопасности

Перед выполнением скрипты проверяют, что:

- они находятся **внутри локального Git-репозитория**
- текущая ветка — `main`
- рабочее дерево чистое
- Git установлен и доступен через `PATH`

Также они создают резервную ветку в таком формате:

    backup-before-teacher-sync-YYYYMMDD-HHMMSS

Если что-то пойдёт не так, предыдущее состояние будет легко восстановить.

---

## 📁 Рекомендуемое расположение

Скрипты можно хранить внутри репозитория, например так:

    MfuaGit/
    ├── myNotes/
    │   └── Script/
    │       ├── sync-from-teacher.sh
    │       └── sync-from-teacher.ps1
    ├── ...

Это работает, потому что оба скрипта автоматически определяют корень репозитория по **местоположению самого файла скрипта**.

## 🐧 Версия для Bash

### Файл

    myNotes/Script/sync-from-teacher.sh

### Сделать исполняемым

    chmod +x myNotes/Script/sync-from-teacher.sh

### Обычный запуск

    ./myNotes/Script/sync-from-teacher.sh

### Запуск с удалением отсутствующих отслеживаемых файлов

    ./myNotes/Script/sync-from-teacher.sh --delete-missing

---

## 🪟 Версия для PowerShell

### Файл

    myNotes/Script/sync-from-teacher.ps1

### Обычный запуск

    powershell -ExecutionPolicy Bypass -File .\myNotes\Script\sync-from-teacher.ps1

### Запуск с удалением отсутствующих отслеживаемых файлов

    powershell -ExecutionPolicy Bypass -File .\myNotes\Script\sync-from-teacher.ps1 -DeleteMissing

---

## 🔍 Разница между обычным режимом и режимом удаления

### Обычный режим

В обычном режиме скрипты:

- копируют файлы преподавателя в мой репозиторий
- перезаписывают совпадающие файлы
- **не удаляют** дополнительные файлы, которые существуют только в моём репозитории

Это более безопасное поведение по умолчанию.

### Режим удаления

В режиме удаления скрипты:

- копируют файлы преподавателя в мой репозиторий
- перезаписывают совпадающие файлы
- удаляют отслеживаемые файлы, которых нет в репозитории преподавателя

Это полезно, когда я хочу, чтобы отслеживаемые файлы в моём репозитории как можно ближе соответствовали репозиторию преподавателя.

> **Важно:** неотслеживаемые файлы **не удаляются**.

---

## 🧠 Как работают скрипты

Процесс простой:

1. определить корень репозитория
2. проверить, что текущая ветка — `main`
3. проверить, что нет незакоммиченных изменений
4. создать резервную ветку
5. получить изменения из репозитория преподавателя
6. скопировать файлы из `teacher/master`
7. при необходимости удалить отсутствующие отслеживаемые файлы
8. подготовить изменения, создать коммит и отправить его

---

## 📝 Пример сообщения коммита

Скрипты создают коммит с таким сообщением:

    Sync files from teacher repo (master)

---

## 🚨 Примечания

Эти скрипты синхронизируют **снимок файлов**, а не полную историю коммитов из репозитория преподавателя.

Это означает, что:

- файлы преподавателя копируются в мой проект
- история моего репозитория остаётся моей собственной
- в моём репозитории коммитятся только итоговые изменения файлов

---

## ✅ Когда это полезно использовать

Эти скрипты полезны, когда мне нужно:

- быстро обновить проект по последней версии преподавателя
- не копировать файлы вручную
- поддерживать порядок в репозитории
- безопасно синхронизировать изменения с резервной веткой
