Dockerfile — это текстовый файл с набором инструкций, из которого Docker собирает образ. Он определяет всё, что будет внутри контейнера: базовую операционную систему, зависимости, код, переменные окружения и команду запуска. Это основа повторяемой и изолированной упаковки приложений.

## 🧱 Основные инструкции Dockerfile

| Инструкция | Назначение | Пример |
|------------|------------|--------|
| `FROM` | Задаёт базовый образ (обязателен, обычно первая строка) | `FROM python:3.11-slim` |
| `WORKDIR` | Устанавливает рабочую директорию (создаёт, если нет) | `WORKDIR /app` |
| `COPY` | Копирует файлы с хоста в контейнер | `COPY . .` |
| `ADD` | Аналогично COPY, но умеет распаковывать архивы и скачивать по URL | `ADD https://example.com/file.tar.gz /tmp/` |
| `RUN` | Выполняет команду во время сборки (установка пакетов, компиляция) | `RUN apt-get update && apt-get install -y curl` |
| `ENV` | Устанавливает переменные окружения | `ENV NODE_ENV=production` |
| `ARG` | Переменные, доступные только во время сборки | `ARG VERSION=latest` |
| `EXPOSE` | Документирует порт, который слушает приложение | `EXPOSE 8080` |
| `CMD` | Команда по умолчанию при запуске контейнера (может быть переопределена) | `CMD ["python", "app.py"]` |
| `ENTRYPOINT` | Основная команда, которая не переопределяется при `docker run` (часто используется вместе с CMD) | `ENTRYPOINT ["python"]` + `CMD ["app.py"]` |
| `USER` | Переключает пользователя (рекомендуется для безопасности) | `USER appuser` |
| `VOLUME` | Создаёт точку монтирования для томов | `VOLUME /data` |

## 📦 Пример Dockerfile для Python-приложения (Flask)

```dockerfile
# Базовый образ: лёгкий Python 3.11
FROM python:3.11-slim
# Устанавливаем рабочую папку
WORKDIR /app
# Копируем зависимости (для кэширования слоя)
COPY requirements.txt .
# Устанавливаем зависимости
RUN pip install --no-cache-dir -r requirements.txt
# Копируем весь код
COPY . .
# Документируем порт
EXPOSE 5000
# Запускаем приложение
CMD ["python", "app.py"]
```

## 🔧 Сборка и запуск

```bash
# Сборка образа с тегом myapp
docker build -t myapp .

# Запуск контейнера с пробросом порта
docker run -p 5000:5000 myapp
```

## 💡 Лучшие практики

- **Используйте `.dockerignore`** – исключайте из контекста сборки ненужные файлы (`.git`, `__pycache__`, `node_modules`).
- **Объединяйте команды `RUN`** – чтобы уменьшить количество слоёв и размер образа.
- **Ставьте `COPY` зависимостей отдельно** – чтобы кэшировать их установку, если код меняется часто.
- **Запускайте приложение от непривилегированного пользователя** – добавьте `RUN useradd ...` и `USER ...`.
- **Используйте многоступенчатую сборку** для языков с компиляцией (Go, Rust, Java) – чтобы итоговый образ не содержал инструментов сборки.
- **Выбирайте минимальные базовые образы** (`-slim`, `-alpine`), когда это возможно.

## 🧠 Как это работает

Каждая инструкция создаёт новый **слой** в образе. Docker кэширует слои, поэтому если вы не меняли `requirements.txt`, слой с установкой зависимостей будет взят из кэша, что ускоряет пересборку.

## 🎯 Заключение

** — это главный инструмент для превращения вашего кода в переносимый, воспроизводимый артефакт. Освоив его, вы сможете упаковывать любые приложения и быть уверенными, что они будут работать одинаково на любой машине с Docker.
