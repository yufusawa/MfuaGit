# Жизненный цикл Docker-образа

Жизненный цикл **Docker**-образа включает несколько ключевых этапов — от создания конфигурации до развертывания в production. Каждый этап управляется командами Docker CLI.

## 📋 Этапы жизненного цикла

### 1. Создание Dockerfile
Разработчик создает `Dockerfile` — текстовый файл с инструкциями для сборки образа.

**Ключевые директивы:**
- `FROM` — базовый образ
- `WORKDIR` — рабочая директория
- `COPY` / `ADD` — копирование файлов
- `RUN` — выполнение команд
- `CMD` / `ENTRYPOINT` — команда запуска

### 2. Сборка образа (Build)
```bash
docker build -t my-app:1.0 .
```

**Что происходит:**
- Docker создает слой для каждой инструкции
- Слои кэшируются для ускорения повторной сборки
- Формируется финальный образ с указанным тегом

**Параметры:**
- `-t` — задает тег (имя) образа
- `.` — путь к Dockerfile (текущая директория)

### 3. Теггирование образа (Tagging)
```bash
docker tag my-app:1.0 myregistry.com/my-app:1.0
```

Теггирование:
- Присваивает понятное имя образу
- Подготавливает к публикации в реестр
- Упрощает идентификацию и использование

### 4. Локальное хранение образа
```bash
docker images  # просмотр локальных образов
```

Собранный образ сохраняется в локальном кэше Docker.

**Удаление образа (если не нужен):**
```bash
docker rmi my-app:1.0  # освобождает место на диске
```

### 5. Публикация в реестр

#### Аутентификация
```bash
docker login
# Ввод учетных данных Docker Hub
```

#### Перетегирование для Docker Hub
```bash
docker tag my-app:1.0 username/my-app:1.0
```
- `username` — логин в Docker Hub (обязательно для публикации)
- Рекомендуется семантическое версионирование

#### Загрузка (Push)
```bash
docker push username/my-app:1.0
```

**Доступные реестры:**
- **Docker Hub** — публичный реестр по умолчанию
- **Private registry** — для приватных образов

### 6. Скачивание и использование

#### Загрузка образа (Pull)
```bash
docker pull username/my-app:1.0
```
- Образ скачивается и сохраняется локально
- Становится доступным для всей команды

#### Запуск контейнера
```bash
docker run -d -p 80:80 my-app:1.0
```
- `-d` — запуск в фоновом режиме
- `-p 80:80` — проброс портов (хост:контейнер)
- Создается изолированный экземпляр приложения на основе образа

## 🏆 Best Practices

### Оптимизация сборки
- **Используйте `.dockerignore`** — исключает ненужные файлы
- **Multi-stage builds** — уменьшает размер финального образа
- **Кэширование слоев** — ускоряет повторную сборку

### Безопасность
```bash
docker scan my-app:1.0  # сканирование на уязвимости
```

### Версионирование
```
my-app:latest      # последняя стабильная версия
my-app:1.0         # конкретный мажорный релиз
my-app:1.0-alpine  # версия с указанием базового образа
```

## 📝 Заключение

Жизненный цикл Docker-образа включает:
1. **Конфигурацию** — создание Dockerfile
2. **Сборку** — формирование образа со слоями
3. **Теггирование** — присвоение имени и версии
4. **Публикацию** — загрузка в реестр
5. **Развертывание** — скачивание и запуск контейнеров

Каждый этап обеспечивает гибкость, безопасность и воспроизводимость при работе с контейнеризованными приложениями.
