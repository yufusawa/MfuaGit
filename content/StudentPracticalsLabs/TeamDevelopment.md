## Командная  разработка проекта

> ### Совместная разработка на проекта на git — это процесс, когда несколько разработчиков работают над одним проектом, используя инструменты GitHub для координации и управления изменениями кода.

Преимущества совместной разработки:
* Прозрачность — вся история изменений доступна
* Code Quality — ревью кода улучшает качество
* Knowledge Sharing — коллективное обучение
* Backup — код хранится удаленно
* Координация — меньше конфликтов между разработчиками

> ### Всю работу вести в VDCode и Github

Студентам:
- Создать команды по 3-4 человека, назначить среди них тимлида (главного);
- Тимлид создаёт публичный репозиторий на **Github** с `README.md` и приглашает выбранных соразработчиков для совместного кодинга в Git-сервис в роли **Maintain** (Мейнтейнер);
- Вместе придумывают проект, например - "Сайт компании по разработки Телеграм-ботов";
- Сайт статический, **HTML+CSS+JS** на 5-6 страниц с картинками;
- Договориться между собой в Телеграме, кто какую работу по созданию сайта будет выполнять;
- Создать сайт;
- Красиво оформить `README.md` проекта, с картинками, списками, ссылками и диаграмой на Mermaid и структуру проекта;
- Включить технологию **Github Page** для получения ссылки на сайт;
- Прислать мне ссылки на готовый сайт.


### Оцениваться совместная работа будет по колличеству и качеству коммитов от каждого участника. Буду проверять по `git log -p`
