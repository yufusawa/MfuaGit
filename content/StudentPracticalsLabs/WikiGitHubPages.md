# 📝 ТСамостоятельная работа и Техническое задание: разработка Wiki-сайта с использованием Git и GitHub Pages

## 🎯 Цель проекта
Разработать статический Wiki-сайт на чистом HTML, CSS и JavaScript с организацией кода в репозитории Git и последующим размещением на GitHub Pages.

> Рекомендуется использовать нейросеть для генерации кода веб-страниц

---

## 📋 Основные требования

### 🏗️ Технический стек
- **HTML5** - семантическая разметка
- **CSS3** - стилизация и адаптивный дизайн
- **JavaScript** - интерактивность (допускается использование localStorage)
- **Git** - контроль версий
- **GitHub Pages** - хостинг

### 🌐 Структура сайта (10 страниц)

1. **index.html** - Главная страница
2. **about.html** - О проекте
3. **tutorials.html** - Учебные материалы
4. **examples.html** - Примеры кода
5. **resources.html** - Ресурсы
6. **faq.html** - Часто задаваемые вопросы
7. **contact.html** - Контакты
8. **blog.html** - Блог/Новости
9. **gallery.html** - Галерея проектов
10. **api.html** - API документация

## 🔧 Git workflow

### Ветка `main`:
- ✅ Стабильная версия проекта
- ✅ Только проверенный код
- ✅ Готовая к деплою версия

### Ветка `test`:
- ✅ Экспериментальные функции
- ✅ Новая функциональность
- ✅ Тестовые стили и скрипты

### Процесс работы:
```bash
# 1. Начинаем работу в test ветке
git checkout test

# 2. Разрабатываем новую функцию
# 3. Коммитим изменения
git add .
git commit -m "feat: add dark theme functionality"

# 4. Переключаемся в main
git checkout main

# 5. Мержим изменения из test
git merge test

# 6. Пушим в удаленный репозиторий
git push origin main
```
