### Самостоятельная работа по созданию проекта c привлекательным README.md

**Тема:** «Выполнить привлекательное оформление своего `README.md` проекта, например **"Hello, World!"** на Markdown в **Github**»

Тему можно придумать самому!

**Цель работы:** Сформировать практические навыки качественной работы через контроль версий и практику с использованием современных инструментов (Git, Markdown).

**Общая легенда:**
Студенты выполняют свои проекты на любом современном ЯП (языке программирования), публично размещают его на **Github**.

### Структура проекта

```
helloworld/
├── README.md # Главная страница проекта
├── src/
│   ├── README.md # Только заголовок
├── translations/
│   ├── README.md # Только заголовок
└── resources/ # Для изображений
    └── README.md # Только заголовок
```

Данную структуру можно создать одной командой на Bash:
```shell
mkdir -p src translations/resources && touch README.md src/README.md translations/README.md translations/resources/README.md
```

### Наполните главный файл `README.md`, который находится в корне репозитория, используя **Markdown**-разметку. Он должен содержать:

    *   Заголовки
    *   Навигатор по проекту
    *   Навигатор по документу по якорям
    *   Изображения (2-3 шт.)
    *   Блок с описанием проекта (2-3 предложения) с различными способами выделения текста.
    *   Цитаты
    *   Горизонтальные линии (разделители)
    *   Нумерованный список со ссылками на все созданные документы в папках `src/`, `translations` и `resources`.
    *   Пример форматированного кода
    *   Таблицу сравнения с подобными проектами
    *   Mermaid диаграммы
