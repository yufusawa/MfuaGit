### Самостоятельная работа по созданию красивого README.md проекта

**Тема:** «Выполнить привлекательное оформление своего README.md проекта Hello, World! на Markdown в Git-сервисе»

**Цель работы:** Сформировать практические навыки обеспечения качества через контроль версий работу с использованием современных инструментов (Git, Markdown).

**Общая легенда:**
Студенты выполняют свои проекты на любом современном ЯП (языке программирования), публично размещают его на любом популярном Git-сервиск для презентации всем заинтересованным.

### Примеры

[Lazyjournal](https://github.com/darthnorse/dockmon/)

[GitType](https://github.com/unhappychoice/gittype?tab=readme-ov-file)

### Структура проекта

```
helloworld/
├── README.md # Главная страница проекта
├── src/
│   ├── README.md # Только заголовок
├── translations/
│   ├── README.md # Только заголовок
└── resources/ # Для изображений
    └── README.md # Только заголовок
```

### Наполните главный файл `README.md` в корне репозитория, используя **Markdown**-разметку. Он должен содержать:

    *   Изображения
    *   Блок с описанием проекта (2-3 предложения).
    *   Нумерованный список со ссылками на все созданные документы в папках `src/`, `translations` и `resources`.
    *  Пример кода
    *  Таблицу сравнения с подобными проектами
