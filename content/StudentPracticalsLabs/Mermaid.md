### **Уровень 2: Средний (Комплексные диаграммы)**

#### **Задание 2.1: Диаграмма классов UML**
**Цель:** Понимание ООП и отношений между классами.

```markdown
Создайте диаграмму классов для "Библиотечной системы":
- Класс `Book` с полями: title, author, ISBN, isAvailable
- Класс `User` с полями: name, userId, borrowedBooks[]
- Класс `Library` с полями: books[], users[]
- Отношения:
  - User *-- Book (агрегация)
  - Library o-- Book (композиция)
  - Library o-- User (композиция)
```
Примерный результат

![Примерный результат](/content/img/mermaid_4.png)

Ключевые элементы синтаксиса диаграмм классов:
Модификаторы доступа:
- `+` - public
- `-`  - private
- `#` - protected
- `~` - package/internal

Типы отношений:
- `*--` - агрегация (часть может существовать отдельно)
- `o--` - композиция (часть не может существовать отдельно)
- `<|--` - наследование
- `-->` - ассоциация

Мультипликаторы:
- `"1"` - ровно один
- `"0..*"` - от нуля до многих
- `"1..*"` - от одного до многих

Типы данных:
- List~Book~ - дженерик тип
- Book[] - массив
- String, Boolean, Integer - примитивные типы
