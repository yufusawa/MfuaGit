## DevOps

### CI/CD - пайплайн (конвейер)

- **CI** (Непрерывная интеграция): Вы пушите код в репозиторий → GitHub автоматически запускает сборку и тесты, чтобы убедиться, что код рабочий .
- **CD** (Непрерывная доставка/развертывание): Если тесты прошли, код можно автоматически выкатить (например, на сервер или в облако)

* `Workflow` (Рабочий процесс): Сам файл с инструкциями, который вы напишете.
* `Job` (Задача): Например, "Запустить тесты" или "Собрать Docker-образ".
* `Step` (Шаг): Конкретная команда внутри задачи, например, npm install.
* `Runner`: Это виртуальная машина, которая выполняет ваши задачи. GitHub предоставляет её бесплатно.
